---

### DevOps
* Base de donnée: [PostgreSQL](https://www.postgresql.org/)

* [AWS](https://aws.amazon.com/)

* Langage: [PHP Symfony 7.2](https://symfony.com/) : framework et langage avec lesquels l’équipe à le plus d’expérience, convient au besoin d’une API Rest. Pas forcément le plus adapté pour du mobile asynchrone par rapport à du JS par exemple mais convenable quand même.

### Frontend
* Application Mobile & Web: [React Native 0.77.0](https://reactnative.dev/) (latest)
* Site Vitrine: [NextJS 15.1.6](https://nextjs.org/) (latest), [Tailwind 4.0](https://tailwindcss.com/), [Lucide](https://lucide.dev/)
* Librairies: Shadcn, HeroUI, RNR, Zustand
* CMS: [EasyAdmin](https://symfony.com/bundles/EasyAdminBundle/current/index.html) (Symfony & [Twig](https://twig.symfony.com/))
* Normes: [ESLint](https://eslint.org/) pour NextJS et React Native, [TwigCS](https://github.com/friendsoftwig/twigcs) pour le CMS

## Gestion de version

---

* Outil: Gitlab

### Nommage des branches
> **release**/fonctionnalité-groupée

> **feat**/nouvelle-fonctionnalité

> **fix**/correctif-léger

> **chore**/tache-pénible

> **hotfix**/correctif-urgent

### Nommage des commits
> **feat**: ma nouvelle fonctionnalité

> **fix**: la fonction trie maintenant en sens inverse

> **chore**: typo dans un texte, refactoring d’une fonction

> **hotfix**: modif cors pour que l’auth marche

<Warning>
    * Taggage des commit sur __master__ pour versionner: **(0.5; 1.0)**
    * Faire des **merge request**
    * Mentionner le ticket associé à sa feature sur la **merge request**
</Warning>

---

### Couverture de test
<Check>**100%** - Frontend & Backend</Check>

### Types de tests
* Unitaires: vérifient une fonction ou composant isolé
* Intégration: vérifient si plusieurs modules fonctionnent ensemble
* Fonctionnels: vérifient que le système répond aux attentes
